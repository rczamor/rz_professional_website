Article Design System

Every component used in the /thinking article prototypes, documented with live examples and usage guidance. Place this file in the website repo for reference by future systems.

Usage rules

Primary accent is always site indigo (#7B7FE4). Do not replace it with semantic colors for UI chrome (nav, progress bars, section labels, CTAs).

Green, amber, and red are semantic only: data visualization phases, medical/EKG themes, error states, breaking-news badges, terminal syntax highlighting. Never use as a primary accent.

The --accent-dim token is indigo at ~15% opacity. Use for hover fills, active node backgrounds, selected button states.

Borders use var(--border-subtle) which is white at 6% opacity. Hover borders use var(--border-hover) at 12%.

All token names match globals.css in the Next.js repo. When integrating, reference var(--bg-primary), var(--text-secondary), etc. directly.

02 / Foundation

Typography

Three typefaces. Bricolage Grotesque for all headings (h1, h2, h3). Inter for body and deck text. JetBrains Mono for technical, UI, and label text. No decorative or serif fonts ever. Matches the main site's font stack.

Usage rules

Only Inter and JetBrains Mono. No Instrument Serif, no decorative fonts, no script fonts.

Display headlines use clamp() for fluid scaling. The first value is the mobile floor, the last is the desktop cap.

JetBrains Mono is for: navigation, eyebrows, section labels, meta bars, code, terminal content, button labels, badge text, chart labels, and axis labels.

Inter is for everything else: headlines, body prose, deck subtitles, card titles, pullquotes.

Negative letter-spacing on headlines (-0.02em to -0.03em) is load-bearing. Don't remove it.

05 / Content

Story Block (Opening Narrative)

A full-width section with darker background for the opening personal story. Uses a large accent-colored dropcap on the first letter.

The first AI product I shipped at Grandstage worked in the demo and failed in the wild. Same model. Same prompts. Same team. What changed was the data.

That gap is what I want to write about. It isn't a model gap. It's a context gap.

When to use

Use once per essay, immediately after the hero. Opens with a personal moment or specific failure.

Background is var(--bg-secondary) (#0a0a0f) with top and bottom var(--border-subtle) borders.

First letter of first paragraph floats left at 64px, 700 weight, accent-colored.

Max-width 760px. Slightly wider than prose (680px) for a more open reading rhythm.

06 / Content

Pullquote

Centered large-format quote. Use only real one-liners from the established rotation.

Retrieval without consolidation is just efficient delivery of noise.
from the rotation

Usage rules

Only use quotes from the real rotation. Never invent pullquotes.

Approved rotation: "Retrieval without consolidation is just efficient delivery of noise." / "Your AI doesn't have a model problem. It has a context problem." / "Expert decision-makers use less information, not more. Your AI should too." / "Models get smarter every month. Context quality stays the same."

Maximum one pullquote per essay. Place it at the structural midpoint, between the analytical body and the product outcome.

The cite attribution uses mono, dim color, normal (not italic) style.

07 / Content

Close / Takeaway

The final content section. A mono kicker, a bold headline statement, and one paragraph of supporting prose.

The takeaway

If you're building AI and your quality ceiling is lower than you want, look at your context layer before you look at your model.

Your AI doesn't have a model problem. It has a context problem.

Usage rules

Every essay ends with this pattern. Kicker is always "The takeaway" in accent-colored mono.

Headline is a single actionable statement. Not a summary. Not a question. A direct instruction the reader can act on.

The supporting paragraph is short (1-2 sentences). Muted color. No performative closers ("Thanks for reading," "Stay tuned").

Sits above the footer with a border-top: 1px solid var(--border-subtle) separator.

Dyptich (Analogy Panel)

Two-panel comparison with no visual hierarchy between them. Left panel is card-colored, right is bg-2. Use for analogies between domains.

Cardiology

Not which test exists. Which one will change the decision.

AI product

The good product knows which chunk to retrieve.

Not which chunk is related. Which one will change the answer.

When to use

Use for domain analogies where two fields illuminate each other. Stack multiple dyptichs vertically for a series of comparisons.

Left panel heading is accent-colored. Right panel heading is muted. This signals "source domain → target domain."

Can be stacked 2-4 times in sequence. More than 4 becomes repetitive.

19 / Interactive

Scrollytelling (Sticky + Steps)

Two-column layout with a sticky visualization on one side and scroll-triggered text steps on the other. The visualization updates as each step enters the viewport.

Architecture

Grid: 1fr 1fr with 80px gap. Sticky column: position: sticky; top: 120px. Steps: min-height: 85vh per step, flex-centered.

IntersectionObserver with rootMargin: '-40% 0px -40% 0px' triggers activation when a step hits the viewport center.

Each step has a data-step attribute. The observer activates nodes/edges where data-step <= current.

Use for the five-step context generation process in the flagship essay. Can be adapted for other multi-stage processes with 3-7 steps.

SVG nodes transition from 25% opacity (inactive) to 100% (active). Edges go from dashed/dim to solid/accent.

On mobile (<900px), the sticky column becomes static and the layout collapses to single-column.

20 / Data Viz

Animated Growth Chart

SVG line chart that draws itself on viewport entry using strokeDashoffset. An area fill fades in after the line completes. Annotation callouts mark inflection points.

Architecture

Path uses stroke-dasharray set to total path length. On IntersectionObserver trigger, stroke-dashoffset transitions from full length to 0 over 2.5s.

Area fill uses a vertical linearGradient from accent at 30% opacity to transparent. It fades in with opacity: 0 → 1 delayed 1s after the line starts.

Annotations are vertical dashed lines with accent circles and mono labels. Only mark real inflection points with qualitative descriptions (e.g. "context v2 ships"), never fabricated numbers.

Use in case study essays. Only with real growth data. Annotations must describe what happened, not made-up metrics.

21 / Data Viz

Live EKG Trace

Continuous animated SVG trace simulating a cardiac monitor. Runs via requestAnimationFrame.

Architecture

The trace is a repeating QRS-complex waveform: flat PR segment, Q dip, sharp R spike, S dip, gentle T bump, flat again. Each "beat" is 100 units wide in SVG viewBox coordinates.

Animation: requestAnimationFrame shifts the offset by 1.5 units per frame. 10-12 beats visible at once. Stroke is accent-colored, 2px.

Container is a card-colored box with a mono title ("Live trace / diagnostic cadence").

Only in the cardiologists essay. A thematic device, not a reusable component.

Reference

Article Assembly Order

The canonical structure for a /thinking essay. Not every component is used in every essay, but the order is always this when a component is present.

Standard article structure

1. Nav (fixed) + Progress bar

2. Hero (eyebrow + display headline + deck + meta) + optional hero attachment (EKG, terminal, big numbers, decay curve)

3. Story block (opening narrative with dropcap)

4. Content sections (section-label + h2.section + prose), interleaved with data display components as needed

5. Pullquote (one per essay, at structural midpoint)

6. Close / Takeaway (kicker + bold headline + one paragraph)

7. Footer

8. Optional: fixed ticker (CNN essay only)

Accent color assignment

All essays use the site accent #7B7FE4 (indigo) for UI chrome: nav, progress bar, section labels, CTAs, card hover borders.

02 Three-phase problem: adds semantic phase colors (--p1 green, --p2 amber, --p3 red) for data visualization only

03 Cardiologists: adds --emerald for EKG trace and medical-themed elements

06 Building week 1: adds semantic terminal colors (--green, --amber, --red) for syntax highlighting

07 CNN interview: adds --amber for CNN brand elements and --red for BREAKING ticker

Copy rules (critical)

No em dashes. Use colons or commas.

No "Context Intelligence" (not an established term).

No Instrument Serif or any decorative font. Headings use Bricolage Grotesque only.

No fabricated numbers. Real proof points only, or mark [TK].

One pullquote per essay, from the approved rotation only.

First person, active voice, contractions. Varied paragraph rhythm.

No "In today's..." openers. No "Moreover/Furthermore." No "game-changing." See rz-copywriting Fatal Fifteen.

No performative closers. The close section's headline is the final word.
